## Critical invariants

**IBKR â€” read-only**: never submit or modify orders. All IBKR modules are read-only.

**options_tracker.csv is derived**: generated by `ibkr_trades.tracker_builder`
from `trades_history.csv`. Never edit manually. The file is gitignored.

**trades_history.csv is append-only**: rows are never deleted or modified after write.
Deduplication key: `trade_id` (IBKR execId / Flex TradeID).

**No async**: the entire system is synchronous. No asyncio, no threads.

**`just positions` vs `just positions-live`**:
- `just positions` â†’ reads local `options_tracker.csv` (rebuilt by last `ibkr-positions` run). Use offline.
- `just positions-live` â†’ reads `reports/output/options_tracker_live.csv` (point-in-time snapshot from last `ibkr-positions` run). Same data source, different file.
- Neither is the "primary" flow anymore â€” `just ibkr-positions` already includes position analysis in its HTML/MD report.

**Dividend tracker invariants**:
- `average_6y` ceiling MUST exclude the current incomplete year
- No technical analysis in dividend decision (BUY / OVERPRICED only)
- TAEE11 replaced EGIE3 on 2026-06-11 â€” do not revert
- PEP `target_weight = 0.0` (monitored via short put, not DCA target)

**IRPF**: no monthly DARF for foreign investments. Only annual IRPF declaration.
PTAX = BCB cotacaoVenda for the trade date (falls back up to 3 prior business days).
