**Status:** Completed
**Skill:** add-feature

**Effort:** S
**Depends on:** T01 (live options export), T02 (risk metrics), task 05-llm-explainer (completed)

---

## Context

`just daily-report-llm` generates an LLM-narrated market analysis using the top
scanner candidates. Currently the LLM has no knowledge of the user's existing
portfolio, so it may recommend adding exposure to AAPL (already 47.4% of NLV),
suggest a new short put when the cash shortfall is $18,276, or recommend a
position already held.

The live IBKR portfolio snapshot (available after `just ibkr-positions`) contains
exactly the context needed to make LLM recommendations portfolio-aware:
- Open positions (to avoid duplicate entries)
- Net delta (to avoid directional overload)
- Cash available (to gate new short puts by buying power)
- Concentration flags (to avoid adding to overlimit positions)

---

## Goal

Pass a compact portfolio summary to the LLM prompt so it can filter and
contextualize scanner recommendations against the user's live portfolio state.

---

## Outcome spec

When done, the following must be true:

1. `just daily-report-llm` accepts an optional `--ibkr-snapshot PATH` flag
   pointing to the JSON or CSV snapshot generated by `just ibkr-positions`.
2. When `--ibkr-snapshot` is provided, the LLM prompt includes a portfolio
   context block containing:
   - Available cash (USD)
   - Open positions: symbol, type, quantity, unrealized P&L %
   - Concentration flags (symbols over limit)
   - Net portfolio delta (from T02, labeled as approximate)
   - Cash shortfall for short puts (if any)
3. The LLM system instruction is extended with:
   `"Do not recommend entering positions already held. Do not recommend cash-
   secured puts if available cash is below worst-case assignment cost. Flag any
   recommendation that increases concentration in a symbol already above 25% NLV."`
4. When `--ibkr-snapshot` is absent, behavior is unchanged from current.
5. No scanner ranking or CSV output is modified.
6. `uv run pytest tests/market_scanner/test_llm_explainer.py` still passes.
7. 2 new tests added for the portfolio context injection.

---

## Constraints

- Portfolio context is injected into the prompt only — never into scanner
  ranking, scoring, or CSV output.
- The snapshot file is read-only. No writes to IBKR.
- Compact format: the portfolio context block must be ≤400 tokens to avoid
  crowding out scanner rows in the prompt. Use abbreviated field names.
- LLM is still optional (the `--llm-explain` flag remains the gate).
  `--ibkr-snapshot` without `--llm-explain` is silently ignored.
- Do not modify `scan.py`, `ranking.py`, `market_state.py`, or any model file.

## Known limitations / follow-up

- The context block is text-only. A future task could pass structured JSON
  to enable the LLM to reason more precisely about margin constraints.
- Sector-level concentration (e.g., "energy sector already at 30%") is not
  included in this task. Would require mapping symbols to sectors.
